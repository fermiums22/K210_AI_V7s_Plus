# Железо верхнего уровня (K210) и фотографии

Здесь идентификация платы и разбор фото со стенда. Сами картинки лежат в
[photos/](photos/) (их надо доложить вручную, см. [photos/README.md](photos/README.md)).

## Состав

| Узел            | Что это                                                                 |
|-----------------|-------------------------------------------------------------------------|
| Вычислитель     | **Sipeed Maix Dock** с модулем **M1W** — SoC **Kendryte K210**: 2×64-bit RISC-V @400 МГц + KPU (нейроускоритель) + APU (аудио) |
| Wi-Fi           | На модуле **M1W** — ESP8285, разъём U.FL под внешнюю антенну (вариант M1 — без Wi-Fi) |
| Дисплей         | 2.4" TFT, шлейф **`Sipeed024-H2441`**, 240×320, контроллер ILI9341-класса, подключение по FPC к разъёму `LCD` на доке |
| Камера          | DVP-модуль на шлейфе **`ZV-T01-GA4.2`** (сенсор GC0328 / OV2640-класс), подключение к камера-FPC дока |
| Накопитель      | microSD **64 GB** (модели kmodel, логи, фото)                            |
| Антенна         | U.FL Wi-Fi антенна с ферритом                                            |
| Питание/прошивка| micro-USB (через USB-serial мост на доке)                               |

K210 хорош тем, что KPU гонит нейросети (детекция лиц/объектов, YOLO) локально, без сервера,
а M1W даёт Wi-Fi — то есть восприятие на борту + канал до Home Assistant.

## Разбор фотографий

> Все три фото сняты на одном столе. Белая изогнутая деталь — верхняя крышка/бампер-кольцо
> робота-пылесоса V7s Plus. Зелёная плата в кадре — родной mainboard пылесоса (нижний уровень),
> его сейчас отдельно запитывают и прозванивают.

### `photos/k210-board-with-lcd-camera.jpg` (фото 1)
Плата Sipeed Maix Dock крупным планом, поставлена вертикально, над ней — 2.4" TFT дисплей на
оранжевом FPC-шлейфе с маркировкой `Sipeed024-H2441` (видны контакты 1…24). К доку подключены:
- слева снизу — **камера** на шлейфе `ZV-T01-GA4.2`;
- по центру — металлический экран K210-модуля, силкскрин `Maix`;
- вставлена **microSD 64 GB** ("smart 64GB");
- по краям — гребёнки GPIO с подписями (IO, 5V, 3V3, GND, SPI/DVP и т.д.).
Это рабочая сборка "K210 + экран + камера" для bring-up.

### `photos/bench-overview.jpg` (фото 2)
Общий вид стенда. Слева — та же сборка K210 + LCD (экран чёрный, не запитан). По центру —
белое бампер-кольцо пылесоса. Справа сверху — **родной mainboard пылесоса** (зелёная плата),
запитан через USB-измеритель и крокодилы (красный/чёрный провод). Поперёк лежит кабель
датчика (термистор/щуп). Показывает, что параллельно идёт работа над нижним уровнем.

### `photos/k210-maix-dock-m1w-wifi.jpg` (фото 3)
Плата вертикально, хорошо читается силкскрин **`SIPEED MAIX DOCK`** и маркировка модуля
**`SIPEED M1W AIoT MODULE K210 A1 INSTD`** с FCC ID. Подключена **U.FL Wi-Fi антенна**
(белый разъём + ферритовый цилиндр на кабеле). Снизу — micro-USB. Над платой — тот же 2.4"
дисплей. Это подтверждает, что у нас именно **M1W (с Wi-Fi)**, а не M1.

## Открытые вопросы по железу
- Точная модель сенсора камеры (GC0328 vs OV2640) — уточнить при первом запуске MaixPy
  (`sensor.get_id()`).
- Какой USB-serial мост стоит на доке (CH55x / другой) — определит нужный драйвер,
  см. [downloads.md](downloads.md).
- Распиновка UART, которым K210 пойдёт к STM32 — согласовать с `../V7s_Plus`.
