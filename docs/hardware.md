# Железо верхнего уровня (K210) и фотографии

Здесь идентификация платы и разбор фото со стенда. Сами картинки лежат в

## Состав

| Узел            | Что это                                                                 |
|-----------------|-------------------------------------------------------------------------|
| Вычислитель     | **Sipeed Maix Dock** с модулем **M1W** — SoC **Kendryte K210**: 2×64-bit RISC-V @400 МГц + KPU (нейроускоритель) + APU (аудио) |
| Wi-Fi           | Есть — модуль помечен **M1W** (силкскрин `SIPEED M1W AIoT MODULE K210`), это Wi-Fi-версия на ESP8285, разъём U.FL. Факт Wi-Fi подтверждён **силкскрином, а не антенной**. У Sipeed бывает и версия **M1 без Wi-Fi** — у нас не она. |
| Антенна (нюанс) | На фото 3 воткнута **не штатная** антенна — внешний штырь с ферритом, добавлен пользователем. Может быть **не на 2.4 ГГц** → риск слабого Wi-Fi. Для боевого режима поставить нормальную 2.4 ГГц U.FL-антенну. На bring-up рядом с роутером не критично. |
| Дисплей         | 2.4" TFT, шлейф **`Sipeed024-H2441`**, 240×320, контроллер ILI9341-класса, подключение по FPC к разъёму `LCD` на доке |
| Камера          | DVP-модуль на шлейфе **`ZV-T01-GA4.2`** (сенсор GC0328 / OV2640-класс), подключение к камера-FPC дока |
| Накопитель      | microSD **64 GB** (модели kmodel, логи, фото)                            |
| Антенна         | U.FL Wi-Fi антенна с ферритом                                            |
| Питание/прошивка| micro-USB (через USB-serial мост на доке)                               |

K210 хорош тем, что KPU гонит нейросети (детекция лиц/объектов, YOLO) локально, без сервера,
а M1W даёт Wi-Fi — то есть восприятие на борту + канал до Home Assistant.

## Подключение к нижнему уровню (STM32) — разъём `JP1`

На STM32-плате (`../V7s_Plus`) распаян разъём `JP1` (2x4, pin 8 = NC) для верхнего
уровня: UART (USART1) + питание 5V/GND + STM32 `Reset`/`BOOT0`. Полная таблица `JP1`
со стороны STM32 — в `../V7s_Plus/README.md`.

Со стороны Maix Dock используем **свободные IO0–IO3** (на доке это пины `D0–D3`).
По официальному `config_maix_dock.py` они ничем не заняты; всё, что нужно для работы
(дисплей/камера/микрофон/аудио/Wi-Fi/USB), сидит на других IO и FPC — их не трогаем.

Распиновка (UART крест-накрест, уровни 3.3 В с обеих сторон → без сдвигателя):

| JP1 (STM32) | цвет | → Maix Dock | K210 IO | роль на Maix |
|---|---|---|---|---|
| 3 | красный | `5V` | — | питание Maix от робота |
| 5, 7 | чёрный | `GND` | — | земля |
| 4 | жёлтый | `D1` | IO1 | UART **RX** ← STM32_TX (PA9) |
| 2 | тёмнокрасный | `D0` | IO0 | UART **TX** → STM32_RX (PA10) |
| 6 | белый | `D2` | IO2 | GPIO-выход → STM32 `BOOT0` |
| 1 | оранжевый | `D3` | IO3 | GPIO-выход → STM32 `NRST` |

`Reset`+`BOOT0` выведены специально: Maix сможет **сам перешивать STM32** по штатному
UART-бутлоадеру (BOOT0=1 → дёрнуть NRST → прошивка через USART1).

Назначение в MaixPy через FPIOA:

```python
from fpioa_manager import fm
from machine import UART
fm.register(0, fm.fpioa.UART1_TX)   # IO0 -> STM32 RX (PA10)
fm.register(1, fm.fpioa.UART1_RX)   # IO1 <- STM32 TX (PA9)
uart = UART(UART.UART1, 115200, timeout=10)
# IO2 = BOOT0, IO3 = NRST — обычные GPIO-выходы
```

### Занятые IO Maix Dock (НЕ использовать для JP1)

| IO | занято |
|---|---|
| IO4, IO5 | REPL/ISP UART (прошивка по USB) |
| IO6, IO7, IO8 | Wi-Fi (ESP8285: TX/RX/EN) |
| IO12, IO13, IO14 | RGB-LED |
| IO9 | **усилитель PAM8403 MUTE** (мод, см. ниже) |
| IO10 | **усилитель PAM8403 SHDN** (мод, см. ниже) |
| IO16 | BOOT-кнопка |
| IO17 | LCD enable (стоковый демо + наш `main.py`) |
| IO20, IO30, IO32 | микрофон (I2S0) |
| IO33, IO34, IO35 | аудио I2S → ЦАП PT8211 |
| IO40–IO47 | камера DVP |
| FPC | дисплей LCD, microSD |

Источник IO-назначений: официальный `config_maix_dock.py` (sipeed/MaixPy-v1_scripts)
и [Maix Dock — Sipeed Wiki](https://wiki.sipeed.com/soft/maixpy/en/develop_kit_board/maix_dock.html).

## Звук: тракт и аппаратный мод против шипения

Тракт воспроизведения: `K210 I2S0 (BCK=IO35, WS=IO33, DA=IO34) → ЦАП PT8211 (U8)
→ усилитель PAM8403 (U7) → динамики (J7/J8)`. Микрофон — отдельный I2S0-вход
(MEMS, DATA=IO20, BCK=IO32, WS=IO30).

**Проблема:** на стоке PAM8403 включён всегда — его `MUTE` (pin 5) и `SHND`/shutdown
(pin 12) подтянуты к +5V (через R21 10K). Программного мьюта нет → усилитель
постоянно шипит (собственный шум класса D + неинициализированный вход ЦАП).
Силенс через I2S шум не убирает — это аппаратная особенность.

**Мод (сделан):** обе управляющие ноги отрезаны от +5V и заведены на свободные
GPIO K210, оба сигнала **active-LOW**, с pull-down 10K (усилитель по умолчанию
ВЫКЛЮЧЕН и не шумит):

| Сигнал PAM8403 | нога чипа | → K210 | логика |
|---|---|---|---|
| MUTE | pin 5 | **IO9** | LOW = мьют |
| SHND (shutdown) | pin 12 | **IO10** | LOW = усилитель выключен (0 шума, 0 тока) |

Прошивка (`src/main.py`): при загрузке держит IO9/IO10 в LOW (тихо), функция
`amp(True)` поднимает оба в HIGH только на время воспроизведения, `amp(False)` —
обратно в тишину.

**Воспроизведение** делается штатным `audio.Audio` + `I2S.play_process()/play()`
(не ручным `I2S.play(buf)` — он неблокирующий и для стрима не годится).
WAV: PCM 16-бит. Пример — `tools/probes/player_test.py`.

## Автозагрузка прошивки (ВАЖНЫЙ подвох)

**MaixPy при вставленной microSD грузится из `/sd`, а не из `/flash`.** После
старта `os.getcwd()` == `/sd`, и автоматически выполняется **`/sd/main.py`**
(если SD нет — тогда `/flash/main.py`). `boot.py` в этой сборке (v0.6.3) не
запускается. `machine.reset()` повторяет этот путь (полный re-init), удобно для
проверки.

Поэтому наш boot-скрипт (`src/main.py` в репо) деплоится на **`/sd/main.py`**:

```
python tools/maix.py COM20 puttext src/main.py /sd/main.py
```

`/sd/main.py` при включении: глушит усилитель (IO9/IO10 → LOW), поднимает Wi-Fi
(креды из `/flash/wifi.json`), печатает IP, оставляет REPL живым.

Запись файлов на плату — только «одним выстрелом» (open+write+close в одном
скрипте); `tools/maix.py puttext/writejson` так и делают. Построчная запись
отдельными raw-REPL командами на этом SPIFFS/SD виснет.
